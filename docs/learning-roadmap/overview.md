# Learning Roadmap

This roadmap records my learning path toward technical writing, documentation engineering, information architecture, content strategy, UX writing, and AI documentation consulting.

## Current phase

I am building a public portfolio while learning the tools and thinking patterns used by modern documentation teams.

## Learning tracks

| Track | Why it matters | Current focus |
| --- | --- | --- |
| Technical writing | Core professional foundation | clarity, structure, task writing |
| Information architecture | Makes large docs usable | navigation, content models, Diataxis |
| Docs-as-code | Connects writing with engineering teams | Git, Docusaurus, GitHub Pages |
| API documentation | Opens developer documentation roles | REST, examples, schemas |
| Structured authoring | Supports enterprise documentation | DITA, Oxygen XML Editor |
| UX writing | Connects documentation with product experience | error messages, onboarding copy |
| AI documentation | Creates future opportunity | AI workflows, knowledge systems |

## Short-term goals

1. Turn this site into a credible job-search portfolio.
2. Add 3-5 polished case studies.
3. Build one API documentation sample.
4. Add real learning notes every week.
5. Prepare role-specific resume language.

## Long-term direction

I want to grow from writer to documentation strategist: someone who can design knowledge systems, improve product communication, and help companies build better documentation workflows with AI.
