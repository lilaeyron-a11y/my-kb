# Portfolio Overview

This portfolio is a public record of my transition into technical writing, documentation engineering, information architecture, content strategy, and UX writing.

The site is built with Docusaurus because the tool itself is part of the story. A documentation portfolio should not only display writing samples. It should show how the writer thinks about structure, navigation, publishing, maintenance, and user journeys.

| Capability | Evidence on this site |
| --- | --- |
| Documentation architecture | A corporate-style knowledge base with onboarding, product docs, support docs, tutorials, reference, and API content. |
| Structured writing | Pages organized around user goals, content types, and reusable patterns. |
| Docs-as-code learning | GitHub Pages, Docusaurus, Markdown, Git, build/deploy workflow, and local development. |
| Technical communication thinking | Notes about Diataxis, DITA, API documentation, UX writing, and AI-assisted documentation. |
