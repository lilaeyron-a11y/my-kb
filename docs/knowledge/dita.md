# DITA

DITA is a structured authoring standard often used for technical documentation.

The important idea is that content can be modular, typed, reusable, and publishable across different outputs.

## Why DITA matters

DITA helps documentation teams manage:

- topic-based writing
- content reuse
- conditional publishing
- consistent structure
- large documentation sets
- multi-output publishing

## What I am learning

I am learning DITA together with Oxygen XML Editor because many enterprise documentation teams still need structured authoring skills.

My goal is not only to know the syntax. I want to understand when DITA is useful, when Markdown is enough, and how structured content changes the writer's thinking.

## DITA and docs-as-code

DITA and docs-as-code are not enemies. They are different answers to different documentation problems.

| Approach | Good for |
| --- | --- |
| DITA | large enterprise docs, reuse, conditional publishing, strict structure |
| Markdown/docs-as-code | developer docs, fast publishing, Git workflows, lightweight sites |

The stronger technical writer should understand both worlds.
