# Technical Writing

Technical writing is the practice of helping people use, understand, and trust technical systems.

It is not only grammar or translation. It includes user research, information architecture, product understanding, content design, writing, editing, publishing, and maintenance.

## My current definition

Technical writing turns complex product knowledge into usable guidance for a specific audience.

The writer needs to understand:

- who the user is
- what task the user wants to complete
- what the user already knows
- what the system can and cannot do
- what risks or mistakes must be prevented
- how the content will be maintained

## Related fields

Technical writing can grow into several related roles:

| Field | Focus |
| --- | --- |
| Information architecture | Organizing content so people can find and understand it. |
| Documentation engineering | Building and maintaining documentation systems and publishing workflows. |
| Content strategy | Planning content according to business goals and user needs. |
| UX writing | Writing interface text that guides product behavior. |
| API documentation | Helping developers understand and integrate with software systems. |
| Knowledge management | Capturing, organizing, and maintaining institutional knowledge. |

## My learning question

How can a technical writer become more valuable in the AI era?

My current answer: by moving from page-level writing to system-level documentation strategy.
