# Docs-as-Code

Docs-as-code means treating documentation like software.

Instead of writing only in a traditional document editor, writers use tools such as Git, Markdown, static site generators, code review, automated checks, and deployment pipelines.

## Why it matters

Docs-as-code is important for modern technical writers because many product teams already work this way.

It helps documentation teams:

- collaborate with engineers
- review content changes in pull requests
- track version history
- publish automatically
- keep docs closer to the product
- automate quality checks

## My current stack

- Markdown for writing
- Docusaurus for publishing
- Git and GitHub for version control
- GitHub Pages for deployment
- Localhost for previewing changes
- npm scripts for build and deploy
