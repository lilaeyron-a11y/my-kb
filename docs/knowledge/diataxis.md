# Diataxis

Diataxis is a documentation framework that separates content by user need.

It is useful because users do not always need the same kind of help. Sometimes they want to learn. Sometimes they want to complete a task. Sometimes they want to understand a concept. Sometimes they only need exact reference information.

## Four content types

| Type | User need | Example |
| --- | --- | --- |
| Tutorial | I want to learn by doing. | Pair your first device. |
| How-to guide | I want to complete a task. | Update firmware. |
| Explanation | I want to understand why. | How wireless pairing works. |
| Reference | I need exact facts. | API parameters or product specifications. |

## Why it matters

Without a framework, documentation can become a messy pile of pages. With a framework, each page has a job.

For a portfolio, Diataxis is also a signal. It shows that the writer can think about documentation as a system, not only as isolated articles.

## How I want to use it

I want to use Diataxis to:

- design knowledge bases
- audit confusing documentation
- decide what kind of page a user needs
- improve navigation labels
- separate conceptual learning from task instructions
