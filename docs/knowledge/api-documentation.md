# API Documentation

API documentation helps developers understand how to integrate with a product or service.

Good API documentation is not only endpoint reference. It also includes authentication, concepts, quickstarts, examples, error handling, limits, SDKs, and troubleshooting.

## Core parts

- Overview
- Authentication
- Quickstart
- Endpoint reference
- Request examples
- Response examples
- Error codes
- Rate limits
- SDK guides
- Changelog

## What makes API docs difficult

API documentation has two audiences:

- humans who need conceptual understanding
- machines and tools that need precise schemas

That means the documentation needs both clarity and strict accuracy.
